## Active LUNARC sessions (2026-05-12 meta-session refresh)

The local `worker-{0..4}.txt` queues are MIGRATED and inert; do not consume them.
All active execution happens inside five LUNARC tmux sessions inside the
holder.sbatch allocation. Each pane reads its initial `/goal` from the matching
`codex-prompts-*.txt` file at the repo root, then consumes follow-up `/goal`
lines from the lane-named queue file under `codex-tasks/<session>/`.

| Session | Panes | Scope | Prompts file | Tasks dir |
|---------|-------|-------|--------------|-----------|
| `nnbar-recon-lunarc` | 7 | Python reconstruction NEXT/audit lanes | `codex-prompts-recon.txt` | `codex-tasks/recon/` |
| `nnbar-sim-lunarc` | 7 | C++ sim, cosmic recovery, thesis-gap kinematics | `codex-prompts-sim.txt` | `codex-tasks/sim/` |
| `nnbar-g4gpu-lunarc` | 4 | G4GPU phases 5/8 + EM-gamma + V7 validation | `codex-prompts-g4gpu.txt` | `codex-tasks/g4gpu/` |
| `nnbar-review-lunarc` | 2 | Geant4 / OpenMC source-review bottleneck DB | `codex-prompts-review.txt` | `codex-tasks/review/` |
| `nnbar-meta-lunarc` | 2 | Project-wide DEBUGGER + VALIDATOR-PLANNER only | `codex-prompts-meta.txt` | `codex-tasks/meta/` |

Total: 22 panes. Planner-recon (recon pane 0), planner-sim (sim pane 0),
and the meta VALIDATOR-PLANNER keep the queues/evidence refreshed. Non-meta
sessions explicitly disable generated debugger/validator panes in
`.codex-supervisor.toml`; the dedicated meta session owns those two roles.
G4GPU and review panes self-iterate per their lane spec until rate-limited.
Heavy runs go via `sbatch` from inside the panes.
