## Wave 2 goal — Part B: seed plan 47 reproduction ledger

Plan 47 specifies the ledger schema (§1), status states (§2), figure
equality protocol (§3), living-document protocol (§4). Wave 2 creates
the actual living ledger:

1. **Create `docs/thesis_reproduction_ledger.md`** with the §1 row
   schema as a header, then add at least **30 ledger rows** drawn
   from chapters 5–10 of the licentiate + PhD theses. Each row has:

   - `id`: e.g. `LIC-CH10-NUM-1`
   - `source`: chapter / section / figure or table reference
   - `thesis_value`: the exact number or "figure" placeholder
   - `reproducing.sample`: dataset id from plan 03 (use stub IDs from
     plans 20–23 for now; leave `not-attempted` status)
   - `reproducing.command`: the CLI invocation that would reproduce

     the inventory pass)
   - `leaf`: the plan 24 leaf id this row exercises
   - `decision_log`: any DEC entries that govern the row (often empty
     until DEC entries land)

   machine-readable YAML form. One row per ledger entry. Use the
