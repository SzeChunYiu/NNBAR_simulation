## Wave 2 goal

Implement the truth-substitution ladder per plan 38, using Wave 1's
`nnbar_reconstruction/statistics/` module for uncertainty propagation.

### Module 1 — `ladder/leaves.py`

A registry of every leaf identifier from plan 24 (V.1, V.2, V.3, V.4,
V.5, C.1–C.6, P.1–P.7, E.1–E.9, S.1–S.6) plus its canonical truth
definition per plan 38 §3. Implementation: a frozen dataclass per
leaf, with fields:

```python
@dataclass(frozen=True)
class Leaf:
    id: str                 # "V.4"
    name: str               # "vertex aggregation"
    branch: str             # "vertex" | "charged" | "photon" | "event_variables" | "selection"
    truth_source: str       # what column/table provides the canonical truth
    output_schema: dict     # {field: dtype}
```

Tests: `test_ladder_leaves.py` confirms every plan-24 leaf has an
entry; per-branch helpers (e.g. `vertex_leaves()`) return the right
subset.

### Module 2 — `ladder/substitute.py`

`substitute_leaf(reco_result, leaf_id, truth_tables)` returns a new
reconstruction-result dict where the named leaf's output column(s)
are replaced by truth values from `truth_tables`. Use the canonical
truth definition from plan 38 §3 — e.g. for V.4 vertex aggregation,
substitute `vertices.vertex_x/y/z` with the mean of true track
foil-projections from `Particle_output`.

Tests: `test_ladder_substitute.py` constructs a minimal reco-result
dict, substitutes V.4, and asserts the output equals the expected
truth value.

### Module 3 — `ladder/run.py`

`run_ladder(output_dir, run, observable, leaves=None) → LadderReport`
runs reco-only, truth-only, and per-leaf truth-at and
truth-everywhere-else modes. Returns the matrix specified in plan
38 §6 with statistical uncertainty per plan 04 §2 bootstrap.

Tests: `test_ladder_run.py` runs the ladder against a tiny synthetic
sample (10 events, fixed seed) and asserts:
- reco-only ≠ truth-only (sanity).
- truth-at-V.4 closer to truth-only than reco-only is (monotone IV).
- ladder is deterministic given a fixed seed.

### Module 4 — `ladder/factorise.py`

`factorise_additive(matrix, leaf_order)` returns the per-leaf
information-value (IV) along the fixed leaf order specified in plan
38 §4. Wave 2 implements only the additive variant; Shapley
permutation averaging is deferred.

Tests: `test_ladder_factorise.py` confirms `Σ IV(L) = obs(truth-only)
- obs(reco-only)` for the additive variant on a synthetic matrix.

### Module 5 — `ladder/cli.py`

CLI entry point. Subcommands:
- `python -m nnbar_reconstruction.ladder run <output_dir> --run <N>
  --observable visible_invariant_mass --json <out.json>`
- `python -m nnbar_reconstruction.ladder factorise <matrix.json>
  --order V.1,V.2,...`

Tests: `test_ladder_cli.py` (optional in Wave 2) — argparse + smoke.
