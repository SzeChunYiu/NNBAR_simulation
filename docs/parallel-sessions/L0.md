## Wave 2 goal

Two-part pass:

### Part A — deepen plan 24 (reconstruction question tree)

For each leaf in §2 (Vertex), §3 (Charged), §4 (Photon/π⁰), §5 (Event
variables), §6 (Selection), populate the per-leaf input/output schema
template that lives in §2.1 currently as a stub. Each leaf needs:

```
Leaf <ID> (e.g. V.4 vertex aggregation)
  inputs (Class A): list of parquet columns from plan 09
  forbidden (Class B): list of truth columns
  decision rule: one paragraph
  output schema: {field: dtype, ...}
  allowed truth use: validation_only / diagnostic_only / labeling
  downstream consumers: subsystem plan IDs
```

The full leaf list is already in plan 24 §2–§6. Plan 09 (deepened in
Wave 1) supplies the column classifications.

### Part B — deepen subsystem plans 25–30 (tracking + charged-PID + vertex)

Each plan currently has the v0.1 skeleton (charter, current
implementation, alternatives, closure, acceptance). Wave 2 adds:

- **Per-leaf input/output schema** matching plan 24's template.
- **Concrete current-implementation citations** — line numbers in
  `nnbar_reconstruction/reconstruction.py` (use plan 08, deepened in
  Wave 1, as the source of truth for line refs).
- **Per-alternative comparison row** — when the plan lists 2+
  alternatives in its "candidates" table, add one row per alternative
  with: source paper / codebase, NNBAR-specific adaptation note,
  expected ladder leaf delta (qualitative until plan 38 lands).
- **Closure-test specification** — turn the §closure paragraph into
  a numbered procedure: dataset id from plan 03, observable, fitter,
  pass criterion.
