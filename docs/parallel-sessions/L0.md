## Working rhythm (per iteration)

1. Re-read `docs/parallel-sessions.md` and this file.
2. Pick the next builder OR SD that lacks detail.
3. Read its source file end-to-end (yes, even the 35 KB ones).
4. Write the sub-section in `07_simulation_atomic_walkthrough.md` under
   the appropriate § with line-number citations.
5. Verify the file is still ≤ 500 lines. If approaching, split — see
   "splitting plan 07" below.
6. `git add docs/rebuild_plans/07_simulation_atomic_walkthrough.md`
7. `git commit -m "docs(plan-07): expand <builder|SD> <name>" -m "Plan: 07" -m "Lane: L0"`
8. `bash /Volumes/MyDrive/nnbar/nnbar/simulation/scripts/codex-supervisor/merge.sh lane/L0-sim-walkthrough`
9. Continue.

## Splitting plan 07

Plan 07 will exceed 500 lines after a few builders are deepened.
When that happens:

- Create `docs/rebuild_plans/07_simulation_atomic_walkthrough/` directory.
- Move §5.4 builder details into per-builder files there:
  `07_5_4_beampipe.md`, `07_5_4_tpc.md`, etc.
- The main `07_simulation_atomic_walkthrough.md` keeps §1–§5.3 plus
  a §5.4 stub that lists each builder with a link to its detail file.
- Same pattern for §6.2 SDs:
  `07_6_2_carbonsd.md`, `07_6_2_tpcsd.md`, etc.
- Each detail file carries its own YAML header with `id` like
  `07_5_4_tpc` and `depends_on: [07_simulation_atomic_walkthrough]`.

The split itself is a separate iteration — commit the split with
message `refactor(plan-07): split for 500-line cap`.
